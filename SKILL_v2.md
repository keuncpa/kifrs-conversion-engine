# K-GAAP → K-IFRS 전환(컨버전) 자동화 스킬 v2

## 개요

한국 기업의 K-GAAP(일반기업회계기준) 감사보고서 및 표준 입력 양식 데이터를 기반으로, K-IFRS(한국채택국제회계기준) 전환에 필요한 종합 분석 엑셀 파일을 **자동 생성**하는 프로세스이다.

v1 대비 변경사항:
- **표준 입력 양식** 연동: `KIFRS_전환_표준입력양식.xlsx`에 회사가 채운 데이터를 자동 파싱
- **자동 매핑 엔진**: 시산표 계정과목에서 K-IFRS 기준서를 자동 매핑
- **자동 산출**: 리스부채, ECL, DBO, 이연법인세 등을 입력 데이터 기반으로 자동 계산
- **정확도 등급**: 입력 데이터 충실도에 따른 결과물 신뢰도 자동 표시
- **적응형 프로세스**: 입력된 시트만 활용, 미입력 시트는 간이 추정으로 대체

---

## 전체 프로세스 흐름 (v2)

```
[입력물 확인]
    ├─ 감사보고서 PDF (필수 or 선택)
    └─ 표준 입력 양식 Excel (선택, 정확도 향상)
        ↓
[STEP 0] 입력 데이터 파싱 및 정확도 등급 산정  ← 신규
        ↓
[STEP 1] 재무제표 엑셀 변환
        ↓
[STEP 2] K-IFRS 전환근거 자동 매핑  ← 고도화
        ↓
[STEP 3] 교차검증 수행
        ↓
[STEP 4] 필요 내부자료 체크리스트 (미입력 시트 기반 자동 생성)  ← 고도화
        ↓
[STEP 5] 전환 변동내역 자동 산출  ← 고도화
        ↓
[최종 엑셀 파일 출력 + 정확도 등급 리포트]
```

## STEP 0: 입력 데이터 파싱 및 정확도 등급 산정 (신규)

### 0-1. 입력물 확인

사용자가 제공한 파일을 확인한다:
1. **감사보고서 PDF** — Read 도구로 읽기 (기존과 동일)
2. **표준 입력 양식 Excel** — openpyxl로 로드, 각 시트 데이터 유무 확인

### 0-2. 표준 입력 양식 파싱

입력 양식이 제공된 경우 아래 순서로 파싱한다:

```python
import openpyxl

wb = openpyxl.load_workbook('KIFRS_전환_표준입력양식.xlsx', data_only=True)

# 각 시트의 데이터 존재 여부 확인
sheets_status = {}
for sheet_name in wb.sheetnames:
    ws = wb[sheet_name]
    has_data = any(
        ws.cell(row=r, column=c).value is not None
        for r in range(2, ws.max_row + 1)
        for c in range(1, ws.max_column + 1)
        if ws.cell(row=r, column=c).fill.fgColor.rgb != 'FFE8F5E9'  # 자동계산 셀 제외
    )
    sheets_status[sheet_name] = has_data
```

### 0-3. 회사 프로파일 추출

회사개요 시트에서 핵심 정보를 추출:
- 업종(대분류) → 자동 가중치 세팅 트리거
- 종속회사 유무 → 연결 의무 판단
- 영업이익 부호 → 손상징후/계속기업 이슈 판단
- 부채비율 → ECL 미래전망 가산율 결정

### 0-4. 정확도 등급 산정

| 등급 | 조건 | 설명 |
|---|---|---|
| A등급 | 12개 시트 중 10개 이상 입력 | 실측 기반, 감사 수준 근접 |
| B등급 | 필수 3개 + 주요 3개 이상 입력 | 주요 항목 실측, 나머지 추정 |
| C등급 | 필수 3개만 입력 (회사개요+TB+COA) | 시산표 기반 추정 |
| D등급 | 감사보고서 PDF만 | 공시 정보 기반 예비 분석 |

## STEP 2: K-IFRS 전환근거 자동 매핑 (핵심 고도화)

### 2-1. 자동 매핑 엔진

`references/auto_mapping_logic.md`의 ACCOUNT_KIFRS_MAP을 참조하여 시산표의 모든 계정과목에 K-IFRS 기준서를 자동 매핑한다.

**처리 순서:**
1. 시산표 각 행의 계정과목명을 ACCOUNT_KIFRS_MAP의 정규식 패턴과 매칭
2. 매칭되면 K-IFRS 기준서, 기본 영향도, 기본 변동사항을 자동 기입
3. 매칭 안 되면 '수동 검토 필요'로 표시

### 2-2. 업종 기반 영향도 재조정

회사개요의 업종에 따라 기본 영향도를 재조정한다:
- 도소매업: 리스(1116), 수익인식(1115) → HIGH 승격
- 제조업: 유형자산(1016), 차입원가(1023) → HIGH 승격
- 건설업: 수익인식(1115, 진행기준) → HIGH 승격
- IT/서비스: 무형자산(1038, 개발비), 수익인식(1115, 라이선스) → HIGH 승격

### 2-3. 추가 시트 데이터 반영

입력된 추가 시트별로 전환근거를 상세화한다:

**리스계약 시트 입력됨 →**
- 계약건수, 연간 총임차료, 총보증금 요약
- 사용권자산/리스부채 예상 산출액 전환근거에 기재
- 변동리스료 존재 시 해당 사항 전환근거에 반영

**임직원정보 시트 입력됨 →**
- DB형 직원 수, 평균 근속연수, 추정 DBO 전환근거에 기재

**금융상품 시트 입력됨 →**
- ECL 산출 결과를 매출채권 전환근거에 기재
- 투자자산 FVOCI/FVPL 재분류 사항 반영

**수익계약 시트 입력됨 →**
- 본인/대리인 판단 결과를 수익 전환근거에 기재
- 변동대가/포인트 프로그램 해당 사항 반영

기존 v1 검증에 추가:

## STEP 4: 필요 내부자료 체크리스트 (고도화)

### 자동 생성 로직

미입력 시트를 기반으로 체크리스트를 **자동 생성**한다:

```
리스계약 시트 미입력 → 리스 관련 자료 요청 항목 자동 추가 (우선순위: 필수)
임직원정보 시트 미입력 → DBO 관련 자료 요청 항목 자동 추가
금융상품 시트 미입력 → ECL 관련 자료 요청 항목 자동 추가
세무정보 시트 미입력 → 세무 관련 자료 요청 항목 자동 추가
... (모든 시트에 대해)
```

### 입력됨/미입력 표시

이미 입력된 시트의 항목은 "✅ 입력 완료"로 표시하고, 미입력 항목만 수집 대상으로 표시한다.

이를 통해 회사가 **추가로 제출해야 할 자료만** 명확하게 보여준다.

## STEP 5: 전환 변동내역 자동 산출 (핵심 고도화)

### 자동 산출 항목

`references/auto_mapping_logic.md`의 산출 규칙에 따라 아래 항목을 자동 계산한다:

#### 리스 (K-IFRS 1116) — 리스계약 시트 활용
```
리스부채 = Σ(월임차료 × 연금현가계수(잔여기간, 증분차입이자율))
사용권자산 = 리스부채 + 보증금할인차금 + 원상복구추정액PV
보증금 현재가치 = 보증금 / (1+r)^n
```

#### ECL (K-IFRS 1109) — 금융상품 시트 활용
```
ECL = Σ(연령구간별 채권잔액 × 과거대손율 × (1 + 미래전망 가산율))
```

#### DBO (K-IFRS 1019) — 임직원정보 시트 활용
```
간이 DBO = Σ(직원별 예상퇴직급여의 현재가치)
순확정급여부채 = DBO - 사외적립자산
```

#### 이연법인세 (K-IFRS 1012) — 세무정보 시트 활용
```
전환 일시적차이 = 리스(ROU-LL차이) + DBO증가 + ECL증가 + 보증금할인
이연법인세 영향 = 일시적차이 × 적용세율
```

### 미입력 시 간이 추정

추가 시트가 미입력인 경우 감사보고서 정보만으로 간이 추정한다 (기존 v1 방식):
- 리스: 임차료 총액 기준 간이 추정
- DBO: 퇴직급여충당부채 × 1.15 (15% 증가 가정)
- ECL: 대손충당금 × 1.10 (10% 증가 가정)

### 시트 구조 (기존 v1 + 정확도 열 추가)

| 계정과목 | K-GAAP 금액 | 전환조정 | K-IFRS 추정 | 변동여부 | 적용 K-IFRS | 변동/무변동 근거 | 추정 가정 | 정확도 |

## 사용자 인터페이스

### 입력 시나리오별 안내

**시나리오 1: 감사보고서 PDF만 제공**
→ 기존 v1 프로세스 실행 (D등급)
→ "표준 입력 양식을 함께 제출하시면 정확도를 높일 수 있습니다" 안내

**시나리오 2: 감사보고서 + 표준 입력 양식 (일부 입력)**
→ 입력된 시트 자동 감지 → 해당 영역 자동 산출
→ 미입력 영역은 감사보고서 기반 간이 추정
→ 정확도 등급 B~C

**시나리오 3: 표준 입력 양식만 (전체 입력)**
→ 감사보고서 없이도 시산표+COA 기반으로 전환근거 생성 가능
→ 정확도 등급 A~B

### 출력물

최종 엑셀 파일에 포함되는 시트:
1. 재무상태표, 손익계산서, 자본변동표, 현금흐름표 (STEP 1)
2. 전환근거 (STEP 2) — 자동 매핑 + 정확도 표시
3. 교차검증 결과 (STEP 3)
4. 필요 내부자료 체크리스트 (STEP 4) — 입력됨/미입력 자동 표시
5. 전환 변동내역 (STEP 5) — 자동 산출 + 정확도 표시
6. **정확도 요약** (v2 신규) — 전체 등급, 항목별 정확도, 개선 권고
