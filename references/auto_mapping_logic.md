# 자동 전환근거 매핑 로직

표준 입력 양식의 데이터를 기반으로 전환근거를 자동 생성하는 규칙이다.

## 1. 입력 데이터 소스별 활용 매트릭스

| 입력 양식 시트 | 활용 대상 STEP | 정확도 향상 영역 |
|---|---|---|
| 회사개요 | 전체 | 업종별 자동 가중치, 연결 판단, 손상징후 |
| 시산표(TB) | STEP 1, 2, 5 | 계정별 금액 정밀 매핑, 변동내역 산출 |
| 계정과목체계(COA) | STEP 2 | K-GAAP→K-IFRS 계정 자동 매핑 |
| 리스계약 | STEP 2, 4, 5 | 리스부채/사용권자산 정확 산출 |
| 임직원정보 | STEP 2, 4, 5 | DBO 추정 정확도 향상 |
| 차입금_사채 | STEP 2, 5 | 유효이자율 산출, 복합금융상품 분리 |
| 금융상품 | STEP 2, 5 | ECL 정확 산출, 금융자산 재분류 |
| 유형무형자산 | STEP 2, 4, 5 | 구성요소 감가상각, 간주원가 |
| 수익계약 | STEP 2, 4 | 본인/대리인, 변동대가 판단 |
| 특수관계자 | STEP 2, 4 | 공시 범위 확대, 보상 공시 |
| 세무정보 | STEP 2, 5 | 이연법인세 정확 재계산 |
| 기타정보 | STEP 2, 4 | 우발부채, 계속기업 판단 |

## 2. 자동 전환근거 생성 규칙

### 2-1. 업종 기반 자동 가중치

회사개요의 '업종(대분류)'를 기반으로 전환 이슈의 영향도를 자동 세팅한다.

```
  LOW:  유형자산(1016), 무형자산(1038)

  HIGH: 유형자산(1016, 구성요소), 차입원가(1023), 재고(1002)

  LOW:  수익인식(1115)

  HIGH: 수익인식(1115, 진행기준), 계약자산/부채

  HIGH: 수익인식(1115, 라이선스), 무형자산(1038, 개발비)
```

### 2-3. 리스 자동 산출 (리스계약 시트 활용)

리스계약 시트의 데이터로 사용권자산과 리스부채를 자동 계산한다.

```
입력: 월임차료, 잔여리스기간, 증분차입이자율, 보증금
산출:
  리스부채 = Σ(월임차료 × 연금현가계수)
  사용권자산 = 리스부채 + 선급리스료 + 원상복구추정액의 현재가치
  보증금 현재가치 = 보증금 / (1 + r)^n
  보증금 할인차금 = 보증금 - 보증금 현재가치 → 사용권자산 가산 또는 선급리스료

증분차입이자율 미입력 시:
  - 차입금_사채 시트의 가중평균이자율 사용
  - 그것도 없으면 업종별 기본값 적용:
    도소매업: 4.5~5.5%
    제조업: 3.5~5.0%
    건설업: 4.0~5.5%
    IT/서비스업: 4.0~5.0%
    기본값: 5.0%
```

### 2-4. ECL 자동 산출 (금융상품 시트 활용)

매출채권 연령분석표 데이터로 ECL을 자동 산출한다.

```
입력: 연령 구간별 매출채권 총액, 과거 대손율
산출:
  각 구간별 ECL = 채권잔액 × 과거대손율 × (1 + 미래전망 가산율)
  미래전망 가산율: 회사 영업이익이 음수이면 +10~20%, 양수이면 +5~10%
  ECL 총액 = Σ(구간별 ECL)
  기존 대손충당금과의 차이 = ECL 총액 - 현행 대손충당금

미래전망 가산율 로직:
  영업이익 > 0, 부채비율 < 200%: 5%
  영업이익 > 0, 부채비율 ≥ 200%: 10%
  영업이익 < 0: 15%
  2기 연속 영업손실: 20%
```

### 2-5. DBO 자동 추정 (임직원정보 시트 활용)

임직원 데이터로 DBO를 추정한다.

```
입력: 임직원명부(입사일, 생년월일, 연봉), 퇴직연금 적립액
산출:
  간이 DBO 추정:
    - 각 직원별 예상퇴직급여 = 현재급여 × (1+임금상승률)^잔여근속기간 × 근속연수/예상총근속연수
    - 총 DBO = Σ(직원별 예상퇴직급여 할인)
    - 순확정급여부채 = DBO - 사외적립자산 공정가치

기본 가정값 (미입력 시):
  할인율: 국고채(AA-) 수익률 기반, 기본 4.0~4.5%
  임금상승률: 업종별
    - 도소매업/서비스업: 3.5%
    - 제조업: 4.0%
    - IT/서비스업: 5.0%
    - 기본값: 4.0%
  이직률: 업종별
    - IT: 15~20%
    - 도소매: 20~30%
    - 제조: 8~12%
    - 기본값: 15%

임직원정보 미입력 시:
  K-GAAP 퇴직급여충당부채 대비 10~20% 증가로 간이 추정
```

### 2-6. 이연법인세 자동 재계산 (세무정보 시트 활용)

전환조정으로 발생하는 새로운 일시적차이에 대한 이연법인세를 산출한다.

```
입력: 일시적차이 명세, 적용 법인세율
산출:
  기존 일시적차이 재검토 + 전환조정 일시적차이 추가:
    - 리스: 사용권자산 vs 리스부채 차이 → 일시적차이
    - DBO 증가분 → 가산할 일시적차이
    - ECL 증가분 → 차감할 일시적차이
    - 보증금 할인 → 일시적차이
  이연법인세 = 일시적차이 × 적용세율

세무정보 미입력 시:
  법인세율 22% 가정 (과세표준 2억~200억 구간)
```

### 2-7. 변동내역 자동 생성 (STEP 5)

위 산출 결과를 종합하여 변동내역 시트를 자동 생성한다.

```
모든 계정과목에 대해:
1. K-GAAP 금액: 시산표에서 추출 (시산표 미입력 시 감사보고서에서 추출)
2. 전환조정: 위 산출 규칙으로 자동 계산
3. K-IFRS 추정: K-GAAP + 전환조정
4. 변동여부: 전환조정 ≠ 0 → '변동', 신규항목 → '신규', 나머지 → '무변동'
5. 근거: 자동 생성 (입력 데이터 기반이면 "입력자료 기반 산출", 추정이면 "감사보고서 기반 추정")
6. 추정 가정: 사용된 이자율, 기간, 가산율 등 자동 기록
```

## 3. 데이터 유무에 따른 정확도 등급

모델은 입력 데이터의 충실도에 따라 결과물의 정확도 등급을 자동 표시한다.

| 등급 | 입력 데이터 | 정확도 | 설명 |
|---|---|---|---|
| A | 전체 시트 입력 | 높음 | 실제 데이터 기반 산출. 감사/검토 수준 근접 |
| B | 필수 시트 + 주요 시트 3개 이상 | 중간 | 주요 항목 실측, 나머지 합리적 추정 |
| C | 필수 시트만 (회사개요+TB+COA) | 기본 | 감사보고서 + 시산표 기반 추정 |
| D | 감사보고서 PDF만 | 예비 | 공시 정보만 활용한 예비 분석 |

각 전환 항목별로도 개별 정확도를 표시한다:
- ★★★ 실측: 해당 시트의 실제 데이터로 산출
- ★★☆ 추정: 관련 데이터 일부 존재, 나머지 업종 평균/가정값
- ★☆☆ 간이: 감사보고서 공시 정보만으로 간이 추정

## 4. 입력 양식 읽기 프로세스

스킬이 입력 양식을 처리하는 순서:

```
1. 입력 양식 Excel 파일 확인
   - openpyxl로 로드
   - 각 시트의 데이터 유무 확인 (빈 시트 vs 입력된 시트)
   - 입력 충실도 등급 산정

2. 회사개요 시트 파싱
   - 업종, 전환기준일, 종속회사 유무, 재무지표 추출
   - 업종 기반 자동 가중치 세팅

3. 시산표 파싱
   - 계정코드/과목명/잔액 추출

4. 추가 시트별 파싱 (입력된 시트만)
```
