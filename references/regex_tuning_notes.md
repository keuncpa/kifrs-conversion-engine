# 자동 매핑 정규식 튜닝 메모

## 발견된 이슈 및 수정사항

### 1. 장기금융상품 오매핑
- 문제: `장기금융상품`이 `재고자산|상품` 패턴에 먼저 매칭됨
- 원인: 패턴 순서가 중요. `장기금융상품` 패턴보다 `상품` 패턴이 먼저 체크됨
- 해결: 매핑 딕셔너리를 리스트로 변경하여 순서 보장, 또는 `상품$` → `^상품$|^재고자산` 등 정확 매칭

### 2. 매출원가 업종 영향도 오조정
- 문제: 도소매업에서 `매출원가`가 `매출` 패턴에 매칭되어 HIGH로 상향됨
- 원인: 업종 가중치의 `매출|수수료매출` 패턴이 너무 넓음
- 해결: 업종 가중치 패턴을 `^상품매출|^수수료매출|^매출$` 등으로 한정

### 3. ECL이 기존 충당금보다 낮게 산출
- 상황: 샘플에서 ECL(128백만) < 현행 대손충당금(150백만)
- 분석: 이론적으로 가능한 경우(기존 충당금이 보수적이었을 때)
- 보완: ECL < 현행 충당금일 때 경고 메시지 추가, "현행 대손율이 이미 보수적 → ECL 전환 영향 경미" 근거 자동 생성

### 4. 정규식 매칭 전략 개선안

**현재**: dict 순회, 첫 번째 매칭 사용
**개선**: 매칭 우선순위 체계

```
1순위: 정확 매칭 (계정과목명 == 패턴)
2순위: 복합어 매칭 (장기금융상품 → "장기" + "금융상품")
3순위: 부분 매칭 (계정과목명에 패턴 포함)
```

또는 계정코드 기반 매핑 추가:
```
1xxx: 자산
2xxx: 부채
3xxx: 자본
4xxx: 수익
5xxx: 비용
```

→ 계정코드가 있으면 코드 기반 1차 분류 후 과목명으로 상세 매핑
